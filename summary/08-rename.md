# 習慣の名前を編集（リネーム）（Issue #21）

追加後に習慣名を変更できるようにした。

## 何ができるようになったか
- 習慣詳細画面の上部で名前を編集・保存できる。
- 閲覧状態では「名前＋編集」、編集状態では入力欄＋保存/取消を表示。
- 空文字・空白のみは保存しない（追加時と同じバリデーション）。
- 保存するとヘッダタイトルや一覧の表示にも即反映（Context 共有のため）。

## 設計
- `HabitsContext` に `renameHabit(id, name)` を追加（trim、空なら無視。保存は既存 useEffect が担当）。
- 詳細画面 `habit/[id].tsx` に編集 state（`editName: string | null`）を持たせ、null=閲覧 / 文字列=編集中で切り替える。

## テスト
- 既存 42 件パス、`tsc --noEmit` エラー無し、Web 詳細 HTTP 200。
  - renameHabit は trim/空無視の単純な不変更新で、既存の純粋関数テストの方針に沿う（UI state のため hook テストは追加せず）。

## スコープ外
- 習慣の並び替え（ドラッグ / 上下移動）は別 Issue 候補
