# 削除前の確認ダイアログ（Issue #19）

✕ の誤タップで習慣が即削除されるのを防ぐため、確認ダイアログを挟んだ。

## 何ができるようになったか
- ✕ タップで即削除せず、「『〇〇』を削除する？」の確認ダイアログを表示。
- 「削除」を選んだときだけ削除。「キャンセル」or 背景タップで閉じる。

## 設計
- React Native の `Modal` で自前の確認ダイアログを実装。
  - react-native-web では `Alert.alert` が動かないことがあるため、クロスプラットフォームに確実な自前モーダルを選択。
- `pendingDelete: Habit | null` の state でダイアログの表示/対象を制御。
- 背景（暗幕）タップでキャンセル。カード本体は伝播を遮断して閉じない。

## スコープ外
- アンドゥ（削除の取り消し）
